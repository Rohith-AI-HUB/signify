C — BASICS & STORAGE CLASSES

Theory Q&A

C vs C++?
C is procedural; C++ is multi-paradigm (procedural + OOP). C++ adds classes, inheritance, polymorphism, templates, STL, references, operator overloading. C is faster to compile, used in embedded/OS kernels. C++ is used in game engines, system software requiring OOP.
Why "middle-level" language?
Combines high-level constructs (loops, functions, data types) with low-level capabilities (pointer arithmetic, direct memory access, bit manipulation). Sits between assembly and high-level languages like Python.
Storage classes? (4 types)
  • auto — default for local vars; stack; lifetime = block scope
  • register — hint to store in CPU register; no address; fastest access
  • static — retains value across calls; local static = file scope; global static = file-only visibility
  • extern — declares var defined in another file; links across translation units
volatile keyword?
Tells compiler: do NOT optimize reads/writes to this variable. Every access goes to actual memory, not a cached register. Critical for:
• Hardware registers (value changes by hardware, not software)
• ISR-shared variables
• Memory-mapped I/O
const keyword?
const int x = 5 — value cannot change after init. const int *p — pointer to const int (can't modify *p). int * const p — const pointer (can't change p). const int * const p — both const.
Local vs Global variables?
Aspect Local Global
Scope Within function/block Entire file / all files (extern)
Storage Stack Data segment (.bss / .data)
Lifetime Until block exits Program lifetime
Default val Undefined (garbage) 0
Static variable?
Retains value between function calls. Initialized once. Stored in BSS/data segment, not stack. 
void counter() {
    static int count = 0;  // initialized once
    count++;
    printf("%d\n", count);   // prints 1, 2, 3... on each call
}
C — POINTERS

Core Concepts

What is a pointer? Why use it?
Variable that stores the memory address of another variable. Uses: dynamic memory allocation, passing by reference (avoid copy), arrays, function pointers, hardware register access.
Explain: int a=10; int *p=&a; — all four: a, &a, p, *p
int a = 10;      // a  → value = 10, stored at some address e.g. 0x100
int *p = &a;    // &a → address of a = 0x100
                // p  → pointer variable, holds 0x100
                // *p → dereference p → value at 0x100 = 10

*p = 20;       // changes a to 20 (same memory location)
NULL pointer?
Pointer initialized to 0 / NULL / nullptr. Points to no valid memory. Always check before dereferencing. Dereferencing NULL → segfault.
Dangling pointer?
Pointer pointing to freed/deleted memory. Causes undefined behavior. 
int *p = (int*)malloc(sizeof(int));
free(p);
// p is now dangling — fix:
p = NULL;   // always nullify after free
Void pointer?
Generic pointer, can hold address of any type. Cannot be dereferenced directly — must cast first. Used in malloc() return, generic data structures. 
void *vp;
int x = 5;
vp = &x;
printf("%d", *(int*)vp);  // cast required
Pointer arithmetic?
Incrementing pointer moves by sizeof(type) bytes, not 1 byte. 
int arr[] = {10, 20, 30};
int *p = arr;
p++;        // moves 4 bytes (sizeof int), points to arr[1] = 20
p + 2;      // points to arr[3] — 8 bytes ahead
p2 - p1;    // difference in elements, not bytes
Double pointer?
Pointer to a pointer. Used in dynamic 2D arrays, modifying a pointer inside a function. 
int a = 10;
int  *p  = &a;   // p  holds address of a
int **pp = &p;   // pp holds address of p
printf("%d", **pp); // 10
Function pointer?
Pointer that stores address of a function. Used in callbacks, dispatch tables. 
int add(int a, int b) { return a + b; }

int (*fp)(int, int) = &add;  // declare & assign
printf("%d", fp(3, 4));      // call → 7
Pointer vs Array?
Array Pointer
Size Fixed at compile time Can point anywhere
sizeof Total array bytes Size of pointer (4/8 bytes)
Reassign Cannot (arr is const ptr) Can change where it points
Memory Contiguous allocation Just stores an address
C — MEMORY MANAGEMENT
Stack vs Heap?
Stack Heap
Managed by Compiler (auto) Programmer (malloc/free)
Size Limited (few MB) Large (limited by RAM)
Speed Fast (LIFO push/pop) Slower (fragmentation)
Lifetime Until scope ends Until explicitly freed
Overflow Stack overflow Memory leak / OOM
Local variables → stack. malloc/calloc → heap.
malloc vs calloc?
int *p = (int*)malloc(5 * sizeof(int));
// Allocates 20 bytes — NOT zero-initialized (garbage values)

int *q = (int*)calloc(5, sizeof(int));
// Allocates 20 bytes — zero-initialized (all 0s)
// calloc takes 2 args: count, size_per_element
realloc?
Resize previously allocated block. Preserves existing data. Returns new pointer (may move block). 
int *p = (int*)malloc(5 * sizeof(int));
p = (int*)realloc(p, 10 * sizeof(int));  // expand to 10 ints
// old data intact; new 5 slots uninitialized
Memory leak? Segmentation fault?
Memory leak: heap memory allocated but never freed. Accumulates over time → program crash / system slowdown. Fix: every malloc needs a matching free.

Segfault: illegal memory access — dereferencing NULL/dangling pointer, accessing out-of-bounds array, writing to read-only memory. OS sends SIGSEGV signal.
Explain line by line: int *ptr = malloc(sizeof(int));
int *ptr = malloc(sizeof(int));
// sizeof(int)     → 4 bytes on most systems
// malloc(4)       → requests 4 bytes from heap, returns void*
// (int*)          → cast to int* (good practice in C++)
// ptr             → pointer variable on stack, stores heap address

*ptr = 42;        // write to heap memory
printf("%d", *ptr); // read from heap
free(ptr);         // release heap block
ptr = NULL;        // prevent dangling pointer
C — STRUCTS & UNIONS
struct vs union?
struct union
Memory Sum of all member sizes + padding Size of largest member only
Access All members simultaneously Only one member at a time (shared memory)
Use case Group related data (person: name+age+id) Type-punning, variant types, memory overlay 
union U { int i; float f; };  // sizeof = 4, not 8
U u; u.i = 10; u.f = 3.14;     // writing f corrupts i
Structure padding?
Compiler inserts padding bytes to align members to their natural alignment (int → 4-byte boundary). Affects sizeof(). 
struct A { char c; int i; };   // sizeof = 8 (3 bytes padding after c)
struct B { int i; char c; };   // sizeof = 8 (3 bytes padding after c)
struct C { char a; char b; int i; }; // sizeof = 8
// Use __attribute__((packed)) to remove padding (may cause perf issues)
C — STRINGS
char array vs string? How stored?
C strings are char arrays terminated by '\0' (null terminator). "hello" stores 6 bytes: h,e,l,l,o,\0. strlen returns 5 (excludes \0), sizeof includes \0.
strcpy vs strncpy? strcmp vs strncmp?
// strcpy  — copies until \0; UNSAFE if dst too small → buffer overflow
strcpy(dst, src);

// strncpy — copies at most n bytes; SAFE but may not null-terminate
strncpy(dst, src, n); dst[n-1] = '\0';  // always null-terminate

// strcmp  — compares full strings; returns 0 if equal, <0 or >0 otherwise
// strncmp — compares first n chars only
strcmp("abc", "abc")  // → 0
strncmp("abcd", "abXX", 2) // → 0 (only first 2 chars)
EMBEDDED C
volatile in embedded — why critical?
volatile uint32_t *GPIO_STATUS = (volatile uint32_t*)0x40020010;
// Without volatile: compiler optimizes repeated reads → reads stale register cache
// With volatile: every read fetches from actual hardware register

volatile int flag = 0;   // shared with ISR
void ISR_Handler() { flag = 1; }
while(!flag) {}          // without volatile, compiler may optimize to infinite loop
Memory-mapped I/O?
Peripheral registers mapped to specific memory addresses. CPU uses normal load/store instructions to control hardware. No special I/O instructions needed (unlike port-mapped I/O). 
// STM32 example: toggle LED via GPIO register at 0x40020014
#define GPIOA_ODR  (*((volatile uint32_t*)0x40020014))
GPIOA_ODR |= (1 << 5);   // set pin 5
GPIOA_ODR &= ~(1 << 5);  // clear pin 5

ISR & Interrupts

What is an ISR? What happens inside an interrupt?
ISR (Interrupt Service Routine): function called by hardware when an interrupt fires.

Sequence:
1. Hardware detects interrupt signal
2. CPU finishes current instruction
3. Saves context (PC, registers, flags) onto stack
4. Jumps to ISR address via interrupt vector table
5. Executes ISR (keep SHORT — no blocking, no malloc)
6. Restores context, resumes normal execution

Interrupt latency = time from interrupt trigger to first ISR instruction. 
void __attribute__((interrupt)) TIMER0_IRQHandler(void) {
    volatile flag = 1;   // set flag, main loop handles work
    TIMER0_SR &= ~0x01;  // clear interrupt flag (mandatory)
}
Rules: no blocking calls, no printf, no malloc inside ISR. Use flags + main loop pattern.
C CODING — ARRAYS & STRINGS
Reverse Array
void reverseArray(int arr[], int n) {
    int l = 0, r = n - 1, tmp;
    while (l < r) {
        tmp = arr[l]; arr[l] = arr[r]; arr[r] = tmp;
        l++; r--;
    }
}
// {1,2,3,4,5} → {5,4,3,2,1}  O(n) time, O(1) space
Reverse String
void reverseStr(char *s) {
    int l = 0, r = strlen(s) - 1;
    while (l < r) { char t = s[l]; s[l++] = s[r]; s[r--] = t; }
}
// "hello" → "olleh"
Count Vowels
int countVowels(char *s) {
    int count = 0;
    for (int i = 0; s[i]; i++)
        if (strchr("aeiouAEIOU", s[i])) count++;
    return count;
}
// "education" → 5
Largest & Second Largest
void findTop2(int arr[], int n) {
    int max1 = INT_MIN, max2 = INT_MIN;
    for (int i = 0; i < n; i++) {
        if (arr[i] > max1) { max2 = max1; max1 = arr[i]; }
        else if (arr[i] > max2 && arr[i] != max1) max2 = arr[i];
    }
    printf("Max: %d, 2nd: %d\n", max1, max2);
}
// {10,50,30,70} → Max:70, 2nd:50   O(n) single pass
Palindrome
int isPalindrome(char *s) {
    int l = 0, r = strlen(s) - 1;
    while (l < r) if (s[l++] != s[r--]) return 0;
    return 1;
}
// "madam" → 1 (palindrome), "hello" → 0
Prime Number
int isPrime(int n) {
    if (n < 2) return 0;
    for (int i = 2; i * i <= n; i++)
        if (n % i == 0) return 0;
    return 1;
}
// O(√n) — check divisors only up to √n
Factorial — iterative & recursive
long long factIter(int n) {
    long long r = 1;
    for (int i = 2; i <= n; i++) r *= i;
    return r;
}
long long factRec(int n) {
    return (n <= 1) ? 1 : n * factRec(n - 1);
}
// iterative preferred in embedded (no stack overflow risk)
Fibonacci — iterative & recursive
void fibIter(int n) {
    int a = 0, b = 1, c;
    for (int i = 2; i < n; i++) { c = a + b; a = b; b = c; }
}
int fibRec(int n) {
    return (n <= 1) ? n : fibRec(n-1) + fibRec(n-2);
}
// fibRec: O(2^n) — avoid for large n; use iterative O(n)
LINKED LIST — ALL OPERATIONS
Node struct + Create + Traverse
struct Node {
    int data;
    struct Node *next;
};

struct Node* createNode(int val) {
    struct Node *n = malloc(sizeof(struct Node));
    n->data = val; n->next = NULL;
    return n;
}

void traverse(struct Node *head) {
    while (head) { printf("%d ", head->data); head = head->next; }
}
Insert at Beginning / End
struct Node* insertFront(struct Node *head, int val) {
    struct Node *n = createNode(val);
    n->next = head;
    return n;   // new head
}

void insertEnd(struct Node *head, int val) {
    struct Node *n = createNode(val);
    if (!head) { head = n; return; }
    struct Node *cur = head;
    while (cur->next) cur = cur->next;
    cur->next = n;
}
Reverse Linked List MOST ASKED
struct Node* reverse(struct Node *head) {
    struct Node *prev = NULL, *cur = head, *nxt;
    while (cur) {
        nxt = cur->next;   // save next
        cur->next = prev;  // reverse link
        prev = cur;        // advance prev
        cur = nxt;         // advance cur
    }
    return prev;           // new head
}
// 1→2→3→4→5 becomes 5→4→3→2→1   O(n) time O(1) space
Find Middle Node (slow-fast pointer)
struct Node* findMiddle(struct Node *head) {
    struct Node *slow = head, *fast = head;
    while (fast && fast->next) {
        slow = slow->next;
        fast = fast->next->next;
    }
    return slow;   // when fast reaches end, slow is at middle
}
// 1→2→3→4→5 → middle = 3
Delete a Node
struct Node* deleteNode(struct Node *head, int val) {
    if (!head) return NULL;
    if (head->data == val) {
        struct Node *tmp = head->next; free(head); return tmp;
    }
    struct Node *cur = head;
    while (cur->next && cur->next->data != val) cur = cur->next;
    if (cur->next) {
        struct Node *tmp = cur->next;
        cur->next = tmp->next;
        free(tmp);
    }
    return head;
}
DSA — STACK & QUEUE
Stack — concept + implementation
LIFO. Applications: function call stack, undo/redo, expression evaluation, DFS. 
#define MAX 100
int stk[MAX], top = -1;

void push(int x) { if (top < MAX-1) stk[++top] = x; }
int  pop()        { return (top >= 0) ? stk[top--] : -1; }
int  peek()       { return (top >= 0) ? stk[top] : -1; }
int  isEmpty()    { return top == -1; }
Queue — FIFO + Circular Queue
FIFO. Applications: BFS, task scheduling, print queue, message buffers. 
#define MAX 100
int q[MAX], front = 0, rear = 0;

void enqueue(int x) { if((rear+1)%MAX != front) { q[rear]=x; rear=(rear+1)%MAX; } }
int  dequeue()     { if(front!=rear){ int v=q[front]; front=(front+1)%MAX; return v; } return -1; }
// Circular: avoids wasted space; (rear+1)%MAX handles wrap-around

Compile-time vs Runtime Polymorphism
Compile-time (overloading): same method name, different parameters. Resolved by compiler. 
int    add(int a, int b)       { return a+b; }
double add(double a, double b) { return a+b; }
Runtime (overriding): child class redefines parent method. Resolved at runtime via vtable. 
class Animal { void sound() { print("..."); } }
class Dog extends Animal { @Override void sound() { print("Woof"); } }
Animal a = new Dog(); a.sound(); // "Woof" — runtime dispatch

Exception Handling — try-catch, throw vs throws, checked vs unchecked

    try {
        FileReader fr = new FileReader(path);
    } catch (FileNotFoundException e) {
        System.out.println(e.getMessage());

EMBEDDED — UART / SPI / I2C
UART — Full form, TX, RX, Baud Rate
Universal Asynchronous Receiver Transmitter
Signal Description
TX Transmit — send data out
RX Receive — receive data in
GND Common ground
Asynchronous: no clock line — both sides agree on baud rate (e.g., 9600, 115200 bps). Start bit + data (5-8 bits) + optional parity + stop bit(s). Simple, 2-wire, point-to-point only.
SPI — MOSI, MISO, Clock, Slave Select
Serial Peripheral Interface — synchronous, full-duplex, master-slave
Line Full form Direction
MOSI Master Out Slave In Master → Slave
MISO Master In Slave Out Slave → Master
SCLK Serial Clock Master → Slave
CS/SS Chip/Slave Select Master → Slave (active low)
Fast (10s of MHz), 4 wires, separate CS per slave. Used for: flash memory, displays, ADC/DAC.
I2C — SDA, SCL, Why only 2 wires?
Inter-Integrated Circuit — synchronous, half-duplex, multi-master multi-slave
Line Full form Description
SDA Serial Data Bidirectional data (open-drain + pull-up)
SCL Serial Clock Clock from master
2 wires because: addressing scheme — each device has a unique 7-bit address sent on SDA. Master sends address → only matching slave responds. Supports 127 devices on same bus. Slower than SPI (100kHz/400kHz/1MHz). Used for: sensors, EEPROMs, RTCs.

ADC / DAC

Why ADC? Why DAC?
ADC (Analog-to-Digital Converter): Real world is analog (temperature, light, sound). MCU only processes digital. ADC converts analog voltage → digital number. Example: NTC thermistor → ADC → temperature value.

DAC (Digital-to-Analog Converter): MCU output is digital. To drive speakers, motors, analog circuits → convert digital value back to analog voltage. Example: PWM/DAC → audio output, motor speed control.

Key specs: Resolution (bits — 12-bit ADC = 4096 steps), Sampling rate (Nyquist: fs > 2 × fmax).
OS CONCEPTS
Deadlock?
4 conditions (Coffman): Mutual Exclusion (resource held exclusively) + Hold & Wait (holds one, waits for another) + No Preemption (can't force release) + Circular Wait (P1→R1→P2→R2→P1).
Prevention: break any one condition (e.g., impose ordering on locks).
Semaphore vs Mutex?
Mutex Semaphore
Type Locking mechanism (binary) Signaling mechanism (counting)
Value 0 or 1 (locked/unlocked) 0 to N
Ownership Thread that locked must unlock Any thread can signal
Use case Protect shared resource (critical section) Control access count / signal between threads
Binary semaphore ≈ mutex but mutex has ownership; semaphore is for synchronization signaling.
Context Switching?
OS saves state (registers, PC, stack pointer) of current process → loads state of next process. Enables multitasking. Overhead: saving/restoring CPU state, cache flush. Triggered by: timer interrupt, system call, I/O wait.
